# Markdown 文档格式规范

## 目的

确保所有生成的 Markdown 文档符合 markdownlint 标准，避免格式警告。

## 核心规则

### 1. 标题格式（MD022）

**规则：** 标题上下必须各有一个空行

✅ 正确：

```markdown
上一段内容

## 标题

下一段内容
```

❌ 错误：

```markdown
上一段内容
## 标题
下一段内容
```

### 2. 列表格式（MD032）

**规则：** 列表前后必须有空白行

✅ 正确：

```markdown
**标题：**

- 列表项 1
- 列表项 2

下一段文字
```

❌ 错误：

```markdown
**标题：**
- 列表项 1
- 列表项 2
下一段文字
```

### 3. 代码块格式（MD031, MD040）

**规则：**

- 代码块前后必须有空白行
- 必须指定语言标识

✅ 正确：

````markdown
上一段文字

```bash
代码内容
```

下一段文字
````

❌ 错误：

````markdown
上一段文字
```bash
代码内容
```
下一段文字
````

**语言标识规范：**

- Shell 命令：```bash
- 纯文本：```text
- INI 配置：```ini
- C 代码：```c
- PowerShell: ```powershell
- Mermaid 图表：```mermaid

### 4. 表格格式（MD060）

**规则：** 分隔线必须有空格，使用紧凑格式

✅ 正确：

```markdown
| 列名 1 | 列名 2 | 列名 3 |
| ------ | ------ | ------ |
| 内容   | 内容   | 内容   |
```

❌ 错误：

```markdown
| 列名 1 | 列名 2 | 列名 3 |
|--------|--------|--------|
| 内容   | 内容   | 内容   |
```

### 5. 标题末尾标点（MD026）

**规则：** 标题末尾不能使用标点符号（包括省略号）

✅ 正确：

```markdown
### 我想…
### 常见问题
### 快速开始
```

❌ 错误：

```markdown
### 我想...
### 常见问题：
### 快速开始！
```

### 6. 链接片段（MD051）

**规则：** 带 emoji 的标题在链接中必须包含 emoji

✅ 正确：

```markdown
## 🚀 快速开始

[快速开始](#-快速开始)
```

❌ 错误：

```markdown
## 🚀 快速开始

[快速开始](#快速开始)
```

### 7. 强调作标题（MD036）

**规则：** 不能使用粗体代替正式标题

✅ 正确：

```markdown
##### 1. 首次配置（一次性）

或者

**答：** 直接回答
```

❌ 错误：

```markdown
**1. 首次配置（一次性）**
```

## 检查清单

在生成任何 Markdown 内容前，必须检查：

- [ ] 所有标题上下有空行
- [ ] 所有列表前后有空行
- [ ] 所有代码块有空行 + 语言标识
- [ ] 所有表格分隔线有空格 `| --- |`
- [ ] 标题末尾无标点
- [ ] 链接包含 emoji（如果标题有 emoji）
- [ ] 不使用粗体代替标题

## 应用范围

此规范适用于所有生成的 Markdown 文档：

- README.md
- 各类指南文档
- 更新日志
- 故障排除文档
- 快速参考卡
- 任何 Markdown 格式输出

## 自动化建议

如果可能，应该在以下环节应用此规范：

1. **生成前检查** - 在输出内容前对照此清单
2. **生成后验证** - 使用 markdownlint 工具检查
3. **持续改进** - 根据新发现的问题更新此规范
