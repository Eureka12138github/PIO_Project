# Markdown 文档生成 Agent

## 职责

负责生成符合 Markdown 格式规范的所有文档内容。

## 核心能力

### 1. 格式规范遵循

- ✅ 自动遵循 `markdown-format-spec.md` 中的所有规则
- ✅ 生成前自动检查格式合规性
- ✅ 避免常见的 markdownlint 警告

### 2. 文档类型

- README.md - 项目概述文档
- 指南文档 - 各类使用指南
- 更新日志 - CHANGELOG.md
- 故障排除 - TROUBLESHOOTING.md
- 快速参考 - QUICK_REFERENCE.md
- 其他 Markdown 文档

### 3. 质量标准

#### 必须满足的规则

1. **MD022** - 标题上下有空行
2. **MD032** - 列表前后有空行
3. **MD031** - 代码块前后有空行
4. **MD040** - 代码块指定语言
5. **MD060** - 表格格式正确
6. **MD026** - 标题无末尾标点
7. **MD051** - 链接片段有效
8. **MD036** - 不使用粗体作标题

#### 输出前检查清单

- [ ] 所有标题格式正确
- [ ] 所有列表有空行
- [ ] 所有代码块有空行 + 语言标识
- [ ] 所有表格使用 `| --- |` 格式
- [ ] 标题末尾无标点符号
- [ ] 链接包含 emoji（如适用）
- [ ] 没有粗体代替标题

### 4. 工作流程

#### 生成新文档

1. 确定文档类型和结构
2. 按照格式规范生成内容
3. 对照检查清单验证
4. 输出最终文档

#### 修改现有文档

1. 读取现有内容
2. 检查格式问题
3. 应用格式规范修复
4. 验证修改后的内容

### 5. 特殊说明

#### 表格格式

始终使用：

```markdown
| 列 1 | 列 2 |
| ---- | ---- |
| 内容 | 内容 |
```

#### 代码块格式

始终使用：

````markdown
上一段文字

```language
代码内容
```

下一段文字
````

#### 列表格式

始终使用：

```markdown
**标题：**

- 列表项 1
- 列表项 2

下一段文字
```
